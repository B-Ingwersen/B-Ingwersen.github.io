Deckless

Web-based multiplayer card games built with Firebase as a backend

This project was inspired by the times friends and I realized we didn't have a deck of cards, or when we were trying to play during a bumpy bus ride in the dark. This aims to provide a simple, web based way to play virtual card games with friends that doesn't include any sign on and offers a decent UI. The project is a work in progress, but you can check out its current state at deckless.web.app

Quick Information

  • Language/Framework: HTML/Javascript, Node.js and Firebase Realtime Database
  • Project Size: ~2,000 Source Lines of Code
  • Features: Frameworks for local and multiplayer games

Overview

Most of the functionality is implemented in the browser, with the backend existing only for facilitating the multiplayer experience. The information passing system is separated out from the visual and game mechanics components, so multiplayer and local games are supported with the same base code. The multiplayer experience is modeled off of sites like Kahoot that use codes for easily creating and joining games.

Game Mechanics

A series of generic game classes are used to store state information about the game (what cards are in which stack, who's turn it is, etc). The game play is driven by processing objects that contain information about each move a player makes. Each distinct game is essentially a move interpreter that defines how to change the game state based on the move information.

Game UI

Most of the graphics are created with Javascript canvases. The visuals are actually split between two canvases, the background one for mostly static elements and the foreground one for elements being animated. The cards are all drawn though basic canvas features (paths, rectangles, text, etc.) meaning there are very few visual resources to load. UI elements like the visual card stacks are linked to the game state information so that actions such as moving a card from one pile to another are represented on screen.

Multiplayer

Information about each multiplayer game is transmitted it by storing it in a Firebase realtime database. This allows the system to find what game to join when the user provides a game code. When the game is actually being played, each player's moves are added as database entries, and each client puts a listener on that database location so that the moves get passed around to everyone.

Local Games

The same base framework is used for local games, but moves are passed straight from where they are generated to what should handle them. Each game can implement a function describing how a computer player should respond to game events.
